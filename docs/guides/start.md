# Общая информарция
Для поддержания работоспособности mesh-сети мы рекомендуем следующие параметры конфигурации в зависимости от назначения вашего узла. Эти параметры обеспечивают приоритет стабильности сети. Пожалуйста, настройте свои узлы в соответствии с этими рекомендациями, чтобы помочь поддерживать здоровую и эффективную сеть для всех участников.

## Рекомендации по настройке

!!! note "Заметка"
    Эти настройки основаны на коллективном опыте сообщества **Mesh-Net Minsk**. Настройки могут изменяться по мере развития сети.

### 📱 1. Персональные мобильные ноды

*Для портативных устройств, устройств, устанавливаемых в транспортных средствах, и обычных пользовательских устройств.*

Большинству пользователей следует использовать эти настройки для узлов, предназначенных для повседневного ношения или установки в транспортных средствах. Эти параметры оптимизированы для мобильности и минимизации нагрузки на сеть, при этом обеспечивая достаточную видимость для других участников.

#### Radio & LoRa

| **Option** | **Recommended Config** | **Notes** | 
| :--- | :--- | :--- | 
| **Lora Region** | `EU_868` | Выбирайте в соотвествии с вашим типом устройства.  | 
| **Device Role** | `CLIENT` or `CLIENT_MUTE` | Используйте `CLIENT_MUTE` если только у вас нет особой необходимости в ретрансляции. | 
| **Modem Preset** | `LONG_FAST` | На конец 2025 года сеть настроена на этот пресет. | 
| **Frequency Slot** | `0` | LongFast по умолчанию (869.525 MHz) для EU_868 пресета. | 
| **NodeInfo Interval** | `10800` (3 hours) | Интервал обновления информации о узле. | 
| **Hop Limit** | `5` | Мы настоятельно рекомендуем использовать значение 5, не более. При установке значения выше 5 вы должны понимать как это влияет на сеть и есть ли в этом необходимость, в противном случае это может привести к перегрузкам сети. | 
| **Ignore MQTT** | `True` | Игнорировать сообщения MQTT в сети Meshtastic. Значие `True` включает игнорирование MQTT пакетов. | 
| **OK to MQTT** | `False` | Разрешить выгрузку данных в MQTT. Значие `False` запрещает выгрузку. | 

#### Position & Telemetry

| **Option** | **Recommended Config** | **Notes** | 
| :--- | :--- | :--- | 
| **Smart Position** | `True` | Умное позиционирование. Обновления вашей позиции будут выходить чаще во время вашего движения. | 
| **Smart Interval** | `15 minutes` | Интервал обновления умной позиции | 
| **Smart Distance** | `100 meters` | Минимальное расстояние, необходимое для повторной передачи ваших данных о местоположении |
| **Broadcast Interval** | `10800` (3 hours)  или `3600` (1 hour) | Вы можете использовать интервал в 1 час если планируете перемещатся в городской сети и хотели бы чтобы участики сети знали более точнее местоположение | 
| **Fixed Position** | `False` | Мобильные узлы должны использовать данные о местоположении GPS/телефона | 
| **Device Metrics** | `Disabled` | Другим участникам сети не нужно знать уровень заряда вашей батареи | 
| **Env. Metrics** | `Disabled` | Не подходит для мобильных устройств |

### 🏠 2. Стационарные/Локальные ноды

*Для домашних базовых станций, узлов на чердаках, мачт на задних дворах и региональных солнечных электростанций средней высоты.*

Поскольку эти узлы стационарны и всегда находятся в сети, им нет необходимости "общаться" по поводу своего местоположения или статуса.

#### Radio & LoRa

| **Option** | **Recommended Config** | **Notes** | 
| :--- | :--- | :--- | 
| **Lora Region** | `EU_868` | Выбирайте в соотвествии с вашим типом устройства.  | 
| **Device Role** | `CLIENT` or `CLIENT_BASE` | Параметр `CLIENT_BASE` помогает гарантировать отправку сообщений от вашего узла, при добавлении остальных домашних узлов в список избранных. | 
| **Is Un-Messagable** | `True` | На узлах, не находящихся под наблюдением, эта функция предотвращает отправку личных сообщений пользователями. | 
| **Modem Preset** | `LONG_FAST` | C конца 2025 года сеть настроена на этот пресет, при расширении количества узлов будет запланировано переключение на другой пресет. | 
| **Frequency Slot** | `0` | LongFast по умолчанию (869.525 MHz) для EU_868 пресета. | 
| **NodeInfo Interval** | `21600` (6 hours) | Стационарные узлы не меняют информацию часто. | 
| **Rebroadcast Mode** | `CORE_PORTNUMS_ONLY` | Игнорируем пакеты с нестандартными номерами портов, такими как TAK, RangeTest, PaxCounter и т. д. Ретранслирует только пакеты со стандартными номерами портов: NodeInfo, Text, Position, Telemetry и Routing. |
| **Hop Limit** | `1-2` | Большое количество переходов нецелесообразно, поскольку обычно это не узел, который получает/отправляет сообщения. | 
| **Ignore MQTT** | `True` | Игнорировать сообщения MQTT в сети Meshtastic. Значие `True` включает игнорирование MQTT пакетов. | 
| **OK to MQTT** | `False` | Разрешить выгрузку данных в MQTT. Значие `False` запрещает выгрузку. | 

#### Position & Telemetry

| **Option** | **Recommended Config** | **Notes** | 
| :--- | :--- | :--- | 
| **Smart Position** | `False` | Узел стационарный, поэтому эта функция не требуется. | 
| **Broadcast Interval** | `86400` (24 hours) | Поскольку узел не перемещается, нет необходимости напоминать сети о своей позиции каждый час. | 
| **Fixed Position** | `True` | Если у устройства нет аппаратного GPS, координаты нужно задать вручную в приложении. | 
| **Device Metrics** | `10800` (3 hours) | Актуально только для узлов с солнечным питанием, иначе можно отключить. | 
| **Env. Metrics** | `10800` (3 hours) | Только если установлены датчики. Используется для проверки утечек в корпусе или перегрева; иначе можно отключить. | 

### Почему эти настройки важны?

!!! warning "Внимание"
    Роли : **ROUTER**, **REPEATER**, **ROUTER_LATE** - Не используйте эти роли без обсуждения с локальным коммьюнити, Вы можете сделать работу сети хуже.

Сеть Meshtastic работает на общем канале со скоростью примерно **1 кбит/с** в режиме *LongFast*. «Болтливые» узлы — те, которые отправляют обновления чаще, чем необходимо, — могут быстро перегрузить сеть, препятствуя передаче реальных сообщений. Увеличивая интервалы и ограничивая количество переходов, мы гарантируем, что сеть останется доступной для всех пользователей.
