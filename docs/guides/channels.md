
# О каналах связи

## Виды каналов связи
Каналы существуют двух типов, публичные и частные, разница лишь в том - каким ключом шифрования общим (публичным) или закрытым (известный только группе лиц) зашифровывалось сообщение.

### Публичные каналы
Публичные каналы видны всем пользователям сети и могут быть использованы для широкого вещания сообщений. Они идеально подходят для общих объявлений, таких как: предупреждения о неблагоприятных погодных явлениях, экстренных сообщений участникам сети, обновления событий или общих сообщений для сообщества. Поскольку они видны всем, важно использовать их для информации, которая может быть полезной для всех участников сети.

В публичных каналах используется общедоступный ключ шифрования `AQ==`, который может быть использован любым участником сети для доступа к этому каналу. С помощью публичного канала происходит обмен информацией и поиск новых устройств (нод).

#### Настройка публичного канала
| **Option** | **Recommended Config** | **Notes** | 
| :--- | :--- | :--- | 
| **Channel Name** | `Primary Channel` | Название публичного канала по умолчанию.  | 
| **Key Size** | `Default` | Использовать шифрование по умолчанию. | 
| **Key** | `AQ==` | Значение ключа шифрованию по умолчанию. | 
| **Channel Role** | `Primary` | Рекомендуемое значение `Primary` для большенства пользователей | 

#### Настройка позиции в публичном канале
| **Option** | **Recommended Config** | **Notes** | 
| :--- | :--- | :--- | 
| **Position Enable** | `True` | Значие `True` разрешает делится локацией. | 
| **Precise Location** | `False` | Для публичных калонов не следует указывать точную позицию| 
| **Approximate Location** | `350m` | Мы рекоменудем устаналивать разброс координат в пределах 350 метров, чтобы остальные участники сети понимаю расположение нод и покрытие сети |

#### Настройка MQTT в публичном канале
| **Option** | **Recommended Config** | **Notes** | 
| :--- | :--- | :--- | 
| **Uplink Unabled** | `False` | Значие `False` запрещается выгрузку данных в MQTT. | 
| **Downlink Enabled** | `False` | Значие `False` запрещается загрузку данных из MQTT. | 

---

### Частные каналы
Частные каналы предназначены для более узкого круга участников и требуют разрешения для доступа. Они идеально подходят для групповых сообщений, таких как координация между членами команды или обмен информацией, которая не предназначена для широкой публики. Частные каналы обеспечивают уровень конфиденциальности и безопасности, позволяя участникам общаться в более контролируемой среде.

### Рекомендации по использованию каналов
- **Публичные каналы**: Используйте для информации, которая может быть полезной для всех участников сети. Избегайте использования для личных сообщений или информации, которая может быть неуместной для широкой аудитории.
- **Частные каналы**: Используйте для координации между членами команды или для обмена информацией, которая не предназначена для широкой публики. Убедитесь, что участники, которым предоставлен доступ, понимают, что информация в этом канале может быть конфиденциальной.
- **Управление каналами**: Регулярно проверяйте и управляйте каналами, чтобы убедиться, что они используются эффективно и что участники имеют доступ только к тем каналам, которые им необходимы. Удаляйте неиспользуемые каналы и обновляйте разрешения по мере необходимости, чтобы поддерживать безопасность и эффективность сети.
