
# Резервное копирование

## Ключи безопасности — резервное копирование и восстановление
Открытый и закрытый ключи будут утеряны и сгенерированы заново при удалении и переустановке прошивки с включенной опцией **полная очистка**. Вы несете ответственность за резервное копирование ключей для поддержания зашифрованной прямой связи с существующими узлами в вашей сети.

!!! info "Информация"
    При изменении короткого имени (short) и длинного имени (long) не требуется смена (перегенерация) ключей, вас идентифицируют по id и публичным ключам другие участиники сети.

### Создание резервной копии
Рекомендуемые методы резервного копирования:
!!! tip "Совет"
    Мы настоятельно рекомендуем установить и использовать консольную утилиту [Meshtastic cli](https://meshtastic.org/docs/software/python/cli/)

* С помощью CLI Meshtastic выполните команду: 

    `meshtastic --export-config > config_backup.yaml` — это сохранит всю вашу конфигурацию, включая открытый и закрытый ключи, в файл.

* Скопируйте ключи в безопасное место, например, в заметку, защищенную паролем.
* Сделайте снимок экрана — это самый простой способ, убедитесь, что вы сохранили все ключи целиком.

### Восстановление из резервной копии
Для восстановления ключей вы можете:

* Использовать CLI Meshtastic для выполнения команды:

    `meshtastic --configure config_backup.yaml` — это самый быстрый способ вернуть весь узел в исходное состояние!

* Вставьте или повторно введите ключи в клиентское приложение из сохраненного места.
